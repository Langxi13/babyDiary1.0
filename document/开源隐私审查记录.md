## 持续门禁

```bash
scripts/privacy-scan.sh
scripts/security-scan.sh
```

CI 对每次推送和拉取请求先获取公开分支、标签、Notes 和 PR refs，再执行隐私披露、依赖、Trivy 和完整历史 Gitleaks 检查。扫描结果只显示类别和文件位置，不回显检测到的私有值。

`config/privacy-host-allowlist.txt` 只允许 RFC 示例、localhost 和明确的公共依赖主机。生产域名、个人网站、服务器 IP、邮箱和部署路径不得通过加入允许列表规避检查。

若凭据曾经进入 Git 历史、Actions 产物或 Release，即使当前扫描不再命中，也必须先轮换或吊销凭据，再清理历史和缓存副本。
