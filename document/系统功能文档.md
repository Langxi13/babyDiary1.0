## 系统概述

### 项目简介

Baby-Diary 是一个面向个人、伴侣和家庭的私有日记系统。系统采用前后端分离架构，既保留个人日记、相册、纪念日等经典功能，也提供共同空间、协作记录、隐私锁、离线同步、AI 总结和可迁移导出。

### 主要特性

- **用户认证系统**：支持短期访问令牌、30天刷新会话、设备管理、邮箱找回和恢复码
- **空间化日记**：自动创建个人空间，并可邀请成员加入共同空间
- **日记管理**：创建、编辑、详情、版本历史、回收站、恢复和乐观锁冲突处理
- **隐私控制**：个人/共同可见范围、密码二次验证日记锁和派生数据脱敏
- **协作互动**：评论、Emoji 回应、站内通知和 Web Push
- **富文本记录**：使用 Tiptap 编辑器支持基础富文本格式，并在后端清洗 HTML
- **标签和心情**：支持自定义标签、固定心情枚举和筛选
- **时间轴和日历**：按月份、周或日期浏览日记，手机端时间轴使用紧凑分组列表
- **纪念日**：维护固定日期、已过天数和倒计时
- **自动相册**：从日记图片生成相册并支持收藏
- **草稿保存**：写日记和编辑日记时自动保存草稿
- **写日记工作台**：正文编辑区和日期、心情、标签、图片等元信息分区展示，发布操作在长内容编辑时保持易触达
- **PWA 安装**：支持安装到桌面或手机主屏，缓存应用壳
- **多图上传**：支持单篇日记上传多张图片
- **图片压缩**：自动压缩大尺寸图片，节省存储空间
- **拖拽上传**：桌面端头像、写日记图片和纪念日封面支持拖拽或点击上传；手机端保留原生选择入口
- **搜索功能**：按日期范围和关键词搜索日记
- **响应优化**：列表页使用轻量摘要、前后端短缓存、批量查询和照片服务端分页减少等待
- **AI 报告**：手动或定时生成周报/月报/年报，把日记文字整理成第三方观察者视角、可使用“你/你们”的 Markdown 回忆总结
- **富媒体与导出**：支持图片、音频、视频、ZIP v2、PDF、EPUB 和跨空间导入
- **离线同步**：共同空间支持 IndexedDB 本地队列、增量同步和冲突提示
- **多端导航**：手机宽度下使用 PWA/Web App Shell，提供固定顶部栏、底部主导航、安全区适配和软键盘避让；桌面端固定展示高频入口，其余入口归入“更多”，避免品牌、功能区和用户区互相遮挡
- **桌面图标**：提供日记本风格 PWA 应用图标，支持添加到手机主屏或桌面后的图标显示

## 用户管理模块

### 功能说明

用户管理模块负责系统的用户认证和授权功能，确保只有授权用户才能访问和管理自己的日记数据。

### 用户注册

**功能描述：**
新用户可以通过邀请码注册账号，系统会验证邀请码的有效性，防止未授权注册。

**输入信息：**
- 用户名（必填，唯一）
- 密码（必填）
- 确认密码（必填，需与密码一致）
- 邀请码（必填，用于验证注册权限）

**业务规则：**
1. 用户名必须唯一，不能与现有用户重复
2. 密码和确认密码必须一致
3. 邀请码必须与系统配置的邀请码匹配
4. 注册成功后，用户需要重新登录才能使用系统

### 用户登录

**功能描述：**
已注册用户通过用户名和密码登录系统，系统验证成功后返回JWT Token，用于后续接口的身份认证。

**输入信息：**
- 用户名
- 密码

**输出信息：**
- 访问令牌（默认15分钟）
- HttpOnly 刷新 Cookie（默认30天并在刷新时轮换）
- Token过期时间和用户基本信息

**业务规则：**
1. 用户名和密码必须匹配
2. 登录成功后创建一条设备会话，可在个人信息页查看并撤销
3. 访问令牌过期后前端自动使用刷新 Cookie 换取新令牌
4. 旧版 `/api/auth/login` 仍返回30天 JWT，仅用于兼容；当前前端使用 V2 登录

### 用户登出

**功能描述：**
用户主动退出登录，清除当前会话的认证信息。

**业务规则：**
1. 登出后当前刷新会话被撤销并清除 Cookie
2. 用户可单独退出某个设备，也可在修改/找回密码后撤销全部设备会话
3. 前端先立即隐藏受保护页面并清除认证信息、API 内存缓存、日记/空间状态、当前空间和二次验证令牌，再异步完成服务端登出
4. API 内存缓存按用户作用域隔离；切换账号后即使请求路径和参数相同，也不会读取上一个账号的纪念日、日记、相册、草稿或其他缓存
5. 旧会话尚未完成的接口响应和刷新令牌结果会被取消，不能在新账号登录后重新写回页面或 Store；其他标签页退出或切换账号时也会同步重置受保护页面

### 获取用户信息

**功能描述：**
获取当前登录用户的详细信息。

**输出信息：**
- 用户ID
- 用户名
- 头像路径
- 邮箱及验证状态
- 系统角色、用户时区和加入时间

### 头像管理

**功能描述：**
用户可以在个人信息页上传头像。头像更新成功后，后端保存新的头像路径并返回最新用户资料；Web 端和手机端在应用启动、页面重新聚焦或从后台回到前台时会刷新当前用户信息，使其他已登录设备尽快同步最新头像。前端在30秒内合并重复刷新，避免切换窗口时重复请求。

**业务规则：**
1. 头像只支持 JPEG、PNG、GIF、WebP，后端同时校验 MIME 和文件签名
2. 头像地址由前端统一图片 URL 工具生成，支持中文、空格等文件名编码
3. 头像上传成功后，当前设备立即更新本地用户资料缓存
4. 其他已登录设备会在下次打开应用或回到前台时刷新用户资料

## 日记管理模块

### 功能说明

日记管理模块是系统的核心模块，提供日记的增删改查、搜索、导出等功能，支持为每篇日记添加多张图片。

### 创建日记

**功能描述：**
用户可以创建新的日记记录，支持添加标题、内容、日期和多张图片。

**输入信息：**
- 日记标题（必填）
- 日记内容（必填）
- 日记日期（必填，格式：yyyy-MM-dd）
- 图片文件（可选，支持多图）
- 内容格式（plain/html）
- 心情标识（可选）
- 标签ID列表（可选）

**业务规则：**
1. 每篇日记必须有标题和内容
2. 图片文件大小限制为单张10MB
3. 支持的图片格式：JPEG、PNG、GIF、WebP
4. 超过300KB的图片会自动压缩
5. 图片压缩参数：最大尺寸1920px，压缩质量0.85
6. 创建成功后，日记自动关联当前登录用户
7. 单篇日记最多接收50张图片、50个标签；列表和 AI 报告分页每页最多100条

### 获取日记列表

**功能描述：**
分页显示用户的日记列表，支持按日期范围、关键词、标签和心情进行筛选。

**输入信息：**
- 开始日期（可选，默认：2022-01-01）
- 结束日期（可选，默认：当天）
- 搜索关键词（可选）
- 标签ID（可选）
- 心情标识（可选）
- 页码（可选，从0开始，默认：0）
- 每页大小（可选，默认：5）
- 轻量摘要开关（可选，`summary=true` 时只返回正文预览片段）

**输出信息：**
- 日记列表
- 总记录数
- 当前页码
- 每页大小

**业务规则：**
1. 只能查看当前登录用户的日记
2. 支持按标题和内容进行关键词搜索
3. 日期范围筛选包含开始和结束日期
4. 每页日记按日期和创建时间倒序排列
5. 首页和列表页默认使用轻量摘要接口，详情页再加载完整正文
6. 关键词输入停止350毫秒后自动筛选；快速切换筛选或页码时，旧请求结果不会覆盖最新结果

### 获取日记详情

**功能描述：**
根据日记ID获取单篇日记的完整信息，包括关联的图片。

**输入信息：**
- 日记ID（必填）

**输出信息：**
- 日记ID
- 用户ID
- 日记标题
- 日记日期
- 日记内容
- 图片路径列表
- 创建时间

**业务规则：**
1. 只能查看自己的日记详情
2. 如果日记不存在，返回错误提示

### 更新日记

**功能描述：**
更新已存在的日记信息，可以修改标题、内容、日期，也可以更新或删除图片。

**输入信息：**
- 日记ID（必填）
- 日记标题（必填）
- 日记内容（必填）
- 日记日期（必填）
- 新图片文件（可选）
- 保留的旧图片文件名（可选）
- 图片显示顺序（可选，支持旧图片和新图片混排）
- 是否清除原有图片（可选，默认：false）

**业务规则：**
1. 只能更新自己的日记
2. 如果选择清除图片，会删除原有的所有图片
3. 新上传的图片同样会自动压缩
4. 编辑日记时，图片使用紧凑缩略图网格展示，可通过上移、下移和删除按钮调整保留图片、新上传图片的顺序
5. 更新后，日记的创建时间保持不变

### 删除日记

**功能描述：**
删除指定的日记及其关联的所有图片。

**输入信息：**
- 日记ID（必填）

**业务规则：**
1. 只能删除自己的日记
2. 删除日记时，会同时删除数据库中的图片记录
3. 删除日记时，会尝试删除服务器上的图片文件；如果单个历史图片文件路径异常或文件清理失败，会记录日志并继续删除数据库记录
4. 删除操作不可恢复，请谨慎操作

### 导出图片

**功能描述：**
将指定日期范围内所有日记的图片打包成ZIP文件导出。

**输入信息：**
- 开始日期（必填）
- 结束日期（必填）

**输出信息：**
- ZIP文件流

**业务规则：**
1. 只能导出自己的日记图片
2. 导出使用服务器随机存储文件名；ZIP 内重名会自动追加序号
3. 如果指定日期范围内没有图片，返回错误提示
4. ZIP文件可以直接下载保存，服务端临时文件会在响应流关闭后自动删除

## 图片管理模块

### 功能说明

系统同时维护旧版图片和 V2 富媒体。旧版图片随日记、头像和纪念日封面管理；V2 媒体支持图片、音频和视频，可绑定共同空间日记，并使用独立对象存储和签名访问地址。

### 上传与存储

1. 支持 JPEG、PNG、GIF、WebP，单张最大10MB。
2. 后端同时校验声明类型和真实文件签名，SVG 或伪造图片会被拒绝。
3. 文件名使用 `diary_{userId}_{uuid}.ext`、`avatar_{userId}_{uuid}.ext`、`anniversary_{userId}_{uuid}.ext` 等服务端随机名称，不保存客户端原始文件名。
4. 新文件会在数据库事务回滚时清理；被替换或删除的旧文件只在事务提交后清理。
5. 日记图片和纪念日封面生成 480px 缩略图，列表与相册优先读取缩略图。
6. V2 媒体支持本地私有目录或 S3 兼容存储，并进行空间额度校验。
7. 可选处理任务使用 FFmpeg/ffprobe 生成视频转码、海报和音频波形，使用 Tesseract 提取图片文字。

### 图片读取

原图通过 `/images/{filename}` 读取，缩略图通过 `/images/thumbs/480/{filename}` 读取，由 Nginx 提供静态文件。文件名不可预测，但图片读取路径本身不要求 JWT，因此不应在日记中上传需要独立访问控制的敏感文件。

V2 富媒体保存在 `DIARY_OBJECT_PATH` 或 S3 中，必须与旧版图片目录隔离。接口只向有权限的成员签发短时 URL；日记锁开启时，查看、修改、删除和新增绑定媒体均需要二次验证。

### 相册归属

收藏、手动相册和 AI 相册只允许引用当前用户日记中的图片。后端在添加或确认相册时重新校验全部 `imageId`，防止跨用户引用。

## 体验增强模块

### 标签和心情

用户可以创建自定义标签，并在日记创建或编辑时选择多个标签。心情使用固定枚举，便于在列表、时间轴、相册中做稳定筛选。

### 时间轴

时间轴按年份、月份和周分层折叠展示日记，支持按年月、标签和心情筛选。进入时间轴或筛选结果刷新后，默认展开当前结果中的全部年份、月份和周；当某个月日记数量达到密集阈值时，会继续按自然周折叠，避免单月记录过多导致页面过长。年、月、周折叠项使用不同的视觉权重和左侧强调线区分层级。每条记录展示标题、日期、标签、心情、内容摘要和最多三张缩略图。手机端时间轴使用更紧凑的分组列表，减少层级缩进，并把日期放入日记卡片内部，提升窄屏阅读效率。

### 日历

日历按月份展示每天的日记数量和首篇标题。点击只有一篇记录的日期会直接进入详情，多篇记录则跳转到日记列表并自动筛选当天。手机端使用日历自身的中文月份标题，上一月、今天和下一月三个操作保持同排，避免重复月份输入框占用空间。

### 纪念日

纪念日支持新增、编辑、删除，展示固定日期、已过天数和倒计时。新增纪念日默认使用用户设备的本地当天日期，避免时区换日造成日期提前。纪念日封面通过上传图片设置，桌面端支持拖拽或点击上传，保存时系统写入 `coverImagePath` 并生成缩略图；用户不需要手动填写或理解服务器文件名。新增和编辑弹窗采用封面区与字段区分栏布局，移动端自动切换为单列，保证标题、日期、描述和操作按钮之间有稳定间距。共同空间设置另提供每日或每周写日记提醒，通过站内通知和已启用的 Web Push 投递。

### AI 报告

AI 报告支持按周、月或年生成 Markdown 回忆总结，可查看、复制和删除历史报告。共同空间所有者可以开启周报、月报或年报定时任务。报告使用第三方观察者视角，可用“你/你们”称呼记录者；锁定日记始终排除在 AI 输入之外。页面包含生成助手、报告预览、历史报告和管理员 AI 配置区域，配置支持 OpenAI 兼容地址、模型列表加载和连接测试。手机端自动切换为单列布局。

### 自动相册

相册自动读取所有日记图片，默认系统相册包含“所有图片”“收藏照片”和按年份聚合的相册。收藏相册基于照片收藏状态动态生成，不复制图片；用户可以在相册详情中收藏或取消收藏单张照片。相册详情首批只读取24张缩略图，用户按需继续加载，页面显示已加载数量和总数；图片等待期间显示骨架占位，读取失败时显示明确的不可用状态。相册组读取失败时保留页面结构并提供重新加载入口。首页收藏照片区只请求展示所需的6张，不再先读取全部收藏图片。

### 富文本和草稿

日记内容使用 Tiptap 富文本编辑器，支持加粗、斜体、引用、无序列表和分隔线。内容以 HTML 格式提交，后端使用 HTML 白名单清洗后保存。写作和编辑过程中自动保存草稿；空白富文本内容不会单独触发草稿保存，发布或更新成功后删除对应草稿。日记图片上传区在桌面端支持拖拽或点击选择，采用紧凑提示区和自定义缩略图网格；编辑日记时可以通过上移、下移按钮调整图片顺序，移动端继续保留原生文件选择入口。

### PWA

前端提供 `manifest.webmanifest` 和 service worker，可安装应用并缓存应用壳。Service Worker 只缓存同源成功响应，只有页面导航在离线失败时回退应用壳；接口、图片和失败响应不会写入壳缓存。共同空间日记可在离线时写入 IndexedDB 队列，联网后按批次推送，并拉取服务端增量变更；版本不一致会进入冲突界面，不会静默覆盖。锁定日记不允许离线创建，媒体队列在恢复网络后上传。

## 共同空间与协作

1. 每个用户注册后自动拥有一个个人空间，也可创建多个共同空间。
2. 共同空间所有者通过限时邀请令牌邀请成员，并可调整角色或移除成员。
3. 日记可设置“共同可见”或“仅自己”；私密日记对其他成员返回不存在，避免泄露元数据。
4. 共同可见日记允许成员协作编辑，使用版本号和 `If-Match` 防止并发覆盖。
5. 日记保留修订快照，可查看历史并恢复；删除先进入回收站，超过保留期后清理。
6. 成员可评论和发送 Emoji 回应，作者可编辑或删除自己的评论，相关事件进入通知中心。

## 隐私安全与账户恢复

1. 日记锁使用密码二次验证，令牌默认10分钟有效。未验证时 V2 列表只返回锁定占位，不返回正文、心情、标签或媒体。
2. 锁定内容不会进入全文索引、搜索结果、AI 输入、年度洞察或离线同步明文载荷；旧版日记、草稿、相册、照片和图片导出接口直接排除锁定条目，防止绕过二次验证。
3. 用户可绑定并验证邮箱，通过邮件链接重置密码，也可保存一次性恢复码用于无邮箱场景。
4. 密码修改或找回会增加 Token 版本、撤销全部刷新会话并清理二次验证令牌。
5. 私密分享支持有效期、访问密码、最多浏览次数和主动撤销；分享链接本身属于敏感凭证。
6. AI 服务的 API Key 使用独立环境密钥加密后入库，配置接口仅管理员可访问。

## 离线同步与数据迁移

1. 客户端为每次离线操作生成幂等操作 ID，服务端保存结果，重复推送不会重复创建日记。
2. 增量同步使用游标分页返回变更；锁定日记只返回脱敏变更。
3. ZIP v2 导出包含日记、标签、评论、旧版图片和 V2 富媒体，并限制文件数量、单项大小和总解压大小。导入锁定日记时，包内旧版图片会写入私有 V2 媒体存储，不会新增公开图片文件。
4. ZIP 导入在跨空间发生 UUID 冲突时确定性重映射，重复导入会跳过已导入记录。
5. 日记书可按条件导出 PDF 或 EPUB；中文 PDF 依赖服务器配置的字体文件。
6. 导出包、备份和 AI 请求都可能包含私密内容，必须按个人数据管理。

## 用户工作流程

所有功能发布前按 [测试与发布验收方案](测试与发布验收方案.md) 验证。自动化覆盖注册登录、日记、共同空间、相册、AI 报告、个人信息和主要路由，并使用 Chromium、Firefox、WebKit 检查浏览器错误与后端 5xx；PWA 安装、系统分享、手机文件选择和安全区仍需 iPhone 与 Android 真机确认。

### 新用户注册流程

```
1. 访问注册页面
2. 输入用户名、密码、确认密码和邀请码
3. 点击注册按钮
4. 系统验证邀请码和用户名
5. 注册成功，跳转到登录页面
6. 使用新账号登录系统
```

### 日常使用流程

```
1. 用户登录系统
2. 查看日记列表
3. 可以进行以下操作：
   a. 点击"新建日记"创建新日记
   b. 点击日记标题查看详情
   c. 点击编辑按钮修改日记
   d. 点击删除按钮删除日记
   e. 使用搜索功能查找特定日记
4. 完成操作后可以选择登出
```

### 创建日记流程

```
1. 点击"新建日记"按钮
2. 填写日记标题
3. 选择日记日期
4. 输入日记内容
5. 拖拽图片到上传区或点击"添加图片"选择图片（可选）
6. 可以预览已选择的图片
7. 点击"保存"按钮
8. 系统自动压缩大图片
9. 保存成功，返回日记列表
```

### 搜索日记流程

```
1. 在搜索框输入关键词（可选）
2. 选择开始日期（可选）
3. 选择结束日期（可选）
4. 点击"搜索"按钮
5. 系统显示符合条件的日记列表
```

### 导出图片流程

```
1. 点击"导出图片"按钮
2. 选择开始日期
3. 选择结束日期
4. 点击"导出"按钮
5. 系统打包图片为ZIP文件
6. 浏览器自动下载ZIP文件
```

### 更新日记流程

```
1. 在日记列表中找到要更新的日记
2. 点击"编辑"按钮
3. 修改日记标题、内容或日期
4. 可以选择是否清除原有图片
5. 可以添加新图片
6. 可以通过图片缩略图上的上移、下移和删除按钮调整图片顺序
7. 点击"保存"按钮
8. 更新成功，返回日记详情
```

### 删除日记流程

```
1. 在日记列表中找到要删除的日记
2. 点击"删除"按钮
3. 系统弹出确认对话框
4. 点击"确认"按钮
5. 删除成功，返回日记列表
```
